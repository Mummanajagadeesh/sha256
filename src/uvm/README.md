# SHA256 UVM Simulation

## Objective

Run and observe a **UVM-based simulation** of a SHA-256 hardware core using ModelSim.
The environment drives several predefined message patterns into the DUT and reports the resulting digests through a UVM scoreboard.

The goal is to:

* compile the RTL and UVM testbench
* execute the simulation using a ModelSim script
* observe digest generation and scoreboard results

All behavior described below is derived from the provided source files and simulation log.

# RTL Modules

Located in `rtl/`.

| File                   | Description                               |
| ---------------------- | ----------------------------------------- |
| `sha256.v`             | SHA-256 top-level wrapper                 |
| `sha256_core.v`        | Core compression logic                    |
| `sha256_k_constants.v` | Constant table used during SHA-256 rounds |
| `sha256_w_mem.v`       | Message schedule memory                   |

These modules together implement the hashing datapath.

# Testbench Components

Located in `tb/`.

The verification environment follows a standard **UVM agent-based structure**.

```
Test
 └─ Env
     └─ Agent
         ├─ Sequencer
         ├─ Driver
         └─ Monitor
```

Observed data is forwarded to a scoreboard for checking.

---

## Top Level

### `tb_top.sv`

Top simulation module.

Responsibilities:

* instantiates the SHA-256 DUT
* instantiates `sha256_if`
* triggers the UVM test using `run_test()`

---

## Interface

### `sha256_if.sv`

Defines the interface between the testbench and the DUT.

Contains the signals used by the driver and monitor to interact with the design.

---

## Transaction

### `sha256_seq_item.sv`

Defines the transaction object used by the sequences.

Important fields:

* `block_in`
* `init`
* `next`
* `mode`
* `expected_digest`

Transactions are generated by sequences and consumed by the driver.

---

## Sequences

### `sha256_sequence.sv`

Base sequence class used for common sequence functionality.

---

### `sha256_sequences.sv`

Defines the sequences used during simulation.

The log shows the following sequences being executed:

| Sequence    | Description                           |
| ----------- | ------------------------------------- |
| `abc_seq`   | Sends the `"abc"` SHA-256 test vector |
| `empty_seq` | Sends an empty message vector         |
| `zero_seq`  | Sends an all-zero message block       |
| `ones_seq`  | Sends an all-ones message block       |
| `multi_seq` | Sends a chained multi-block message   |

Sequences create `sha256_seq_item` objects and send them through the sequencer.

---

## Sequencer

### `sha256_sequencer.sv`

Receives sequence items from the sequence and forwards them to the driver.

---

## Driver

### `sha256_driver.sv`

Receives transactions and drives them onto the DUT through the interface.

Driver responsibilities observed in the log:

* waits for reset release
* sends message blocks
* waits for `digest_valid`
* prints digest information

Example log line:

```
[DRIVER] digest_valid asserted. digest=...
```

---

## Monitor

### `sha256_monitor.sv`

Observes the DUT outputs.

Captured data is forwarded to the scoreboard.

Example log output:

```
[MONITOR] Captured digest=...
```

---

## Scoreboard

### `sha256_scoreboard.sv`

Receives observed digests and compares them with expected values when available.

Scoreboard output seen in the simulation:

```
[SB] PASS [1]
[SB] PASS [2]
[SB] SKIP (no expected set)
```

Final summary printed by the scoreboard:

```
========== SCOREBOARD SUMMARY ==========
PASS : 2
FAIL : 0
========================================
```

# Simulation Script

### `run.do`

ModelSim script responsible for:

1. mapping the `work` library
2. compiling UVM
3. compiling RTL
4. compiling the testbench
5. starting simulation

Simulation is launched with:

```
vsim work.tb_top \
     +UVM_TESTNAME=sha256_test \
     +UVM_VERBOSITY=UVM_MEDIUM
```

The script executes:

```
run -all
```

and then exits the simulator.

---

# Running the Simulation

Run from inside the **tb directory**.

```
vsim -do run.do -batch
```

This command will:

1. execute `run.do`
2. compile all design and testbench files
3. start the simulation
4. run until completion

Inside the simulation the following command is executed:

```
run -all
```
